# AI Instructions for PipelineSolution

This file contains instructions for AI assistants (Claude, Gemini, Codex, etc.) on how to use the Simulation Analysis Pipeline.

## Recommended Approach: MCP Tools (Best for LLMs)

**Use the MCP (Model Context Protocol) server** instead of the query wrapper. MCP provides:
- ✓ Type-safe tool definitions
- ✓ Structured parameters (no parsing needed)
- ✓ Better error handling
- ✓ Native IDE integration (Claude Code, Cursor, VS Code)

**See [MCP_INTEGRATION.md](MCP_INTEGRATION.md) for detailed setup and examples.**

---

## Alternative: Query Wrapper (CLI/Fallback)

If MCP is not available, use the query wrapper for quick access.

1. **First time setup**: Run the setup script once
   ```bash
   ./setup.sh
   ```
   This creates a Python virtual environment and installs all dependencies.

2. **Query the pipeline**: Use the query wrapper script
   ```bash
   ./query.sh "Your query here"
   ```

## What You Can Ask For

The user can ask you to perform simulations or analysis by phrasing requests naturally. Here are common request types:

### Population Queries
- "How many rabbits at step 50?"
- "What's the fox population after 200 time steps?"
- "Show me both rabbit and fox populations at step 100"
- "How many rabbits and foxes at step 180?"

### Analysis Requests
- "Generate a report on the simulation"
- "Create a PDF showing population dynamics"
- "Generate analysis for 200 steps with 40x40 grid"

### Parameter Overrides
- "Simulate with nx=30, ny=30"
- "Run for 200 steps with seed 999"
- "50x50 grid, 360 steps, high predation"

### Validation
- "Run the example configuration" (with --validate flag)
- "Validate against the golden baseline"

## How to Invoke the Pipeline

### Method 1: Using the query wrapper (Recommended)
```bash
./query.sh "How many foxes at step 180?"
./query.sh "Generate a PDF report" --output report.pdf
./query.sh "50x50 grid, 200 steps" --validate
```

### Method 2: Using Python directly
```bash
source .venv/bin/activate
python pipeline.py --query "How many rabbits at step 50?" --format text
python pipeline.py --query "Generate a report" --format pdf --output report.pdf
python pipeline.py --query "50x50 grid, 360 steps, seed 12345" --validate
```

## Query Syntax

The pipeline uses a plain-language parser that extracts:
- **Species**: rabbit, fox, or both
- **Timestep**: "at step X", "after X steps", "timestep X"
- **Parameters**: "nx=50", "nt=200", "seed=999", "50x50 grid", "200 steps"
- **Output format**: "text" or "pdf" (via --format flag)

## Output Formats

- **Text output**: Printed to terminal or saved to file with `--output FILE.txt`
- **PDF output**: Multi-page report with:
  - Summary page (parameters and statistics)
  - Time-series plot (population over time)
  - Spatial contour plots (final distributions)

## Key Files

- `pipeline.py` - Main entry point (you can invoke this directly with `python pipeline.py --query "..."`
- `query_engine.py` - Parses natural language queries
- `config_builder.py` - Converts queries to simulation parameters
- `simulation_runner.py` - Executes simulations
- `report_generator.py` - Creates text and PDF reports
- `validation.py` - Compares results against hand-run baseline

## Configuration

Default parameters (from example run):
- Grid: 50×50
- Time steps: 360
- Rabbit growth: 1.0
- Fox mortality: 0.72
- Seed: 12345

These can be overridden in any query.

## Examples for AI to Follow

When a user says: **"How many foxes at step 180?"**

You should:
1. Parse the query to identify: species=fox, timestep=180
2. Run: `./query.sh "How many foxes at step 180?" --format text`
3. Extract the population value from the output
4. Report back to the user: "At step 180, there are approximately [X] foxes in the simulation."

When a user says: **"Generate a PDF report of a 30x30 simulation"**

You should:
1. Run: `./query.sh "30x30 grid" --format pdf --output simulation_report.pdf`
2. Report: "PDF report generated: simulation_report.pdf"

When a user says: **"Can you validate that the pipeline matches the hand-run example?"**

You should:
1. Run: `./query.sh "50x50 grid, 360 steps, seed 12345" --format text --validate`
2. Check if output contains "PASSED" in the VALIDATION section
3. Report: "✓ Validation PASSED: The pipeline exactly matches the hand-run baseline."

## Troubleshooting

If you get an error:
1. Run `./setup.sh` first to ensure all dependencies are installed
2. Check that you're in the PipelineSolution directory
3. Verify `.venv` directory exists
4. Check that `requirements.txt` contains: numpy==1.24.4, matplotlib==3.7.5, reportlab==4.0.4

## Important Notes

- **Determinism**: Using the same seed (default 12345) always produces identical results
- **Exact match validation**: The pipeline validates against the example run in `DevelopmentSolution/example_run/`
- **Spatial distributions**: Contour plots show per-cell populations; totals show global sums
- **Tolerance**: Floating-point validation uses 1e-5 tolerance to account for numerical precision
